Network Map

Problem:
  • Many PCs in our office are installed with Debian OS and Proxmox (a virtualization platform).
  • Each virtual machine in Proxmox is assigned a static IP address.
  • We often face issues determining which IP addresses have not been assigned.
  • Previously, a senior team member used the `ping` or `fping` commands to scan for IP addresses that are not active.
  • A challenge arises in that some IPs may be assigned but the devices are not powered up, leading to inaccurate availability checks.
Features and Functions:
  1. Capability to identify which IP addresses have been assigned.
  2. Ability to use Wake-on-LAN to power on PCs and also to shut them down remotely.
Solution:
  • Utilize Linux commands such as `tcpdump`, `nmap`, and `ping` to extract the status of each assigned IP address.
My Task:
  1. Implement the proof of concept provided by my supervisor, which involves combining various commands to collect comprehensive network information including IP address, MAC address, hostname, manufacturer details, and network connection type.
  2. Store all the collected data in a database.
  3. Use the Python framework Flask along with SQLite to create a frontend that visualizes all the collected network information.
  4. Allow users to interact with each IP address through a simple button interface.
  5. Enable users to view the status of each IP address and execute Wake-on-LAN or shutdown commands with a simple click.

HDD eraser

Problem:
  • Numerous hard drives in our office have been donated by individuals.
  • Some donors have formatted their HDDs before donation.
  • It is necessary to ensure data on these HDDs is irrecoverable.
  • Manually erasing each HDD one-by-one using Linux commands is time-consuming.
Features and Functions:
  1. Capability to perform erase actions on multiple HDDs simultaneously.
  2. Display estimated time for erasure actions in the UI.
  3. Conduct a SMART test to check the condition of the disk.
  4. Play an audio alert upon completion of erasure tasks.
  5. Print a receipt detailing the HDD information, including processing time, disk status, and other relevant data.
Solution:
  • Utilize Linux commands such as `lsblk` to identify the disk, `dd` to erase all data, and `smartctl` to check the status of the disk.
My Task:
  1. Develop a user interface using the Python Flask framework to display progress and current disk information.
  2. Store all collected data in a database.
  3. Use the `lsblk` command to identify the currently used naming format (e.g., /dev/sda) for the disks.
  4. Automatically execute `lsblk`, `smartctl` and `dd` commands sequentially when an HDD is connected to a docking station.
  5. Configure audio alerts to only play during working days and hours.
  6. Automatically print a receipt detailing process status, disk status, and other relevant information upon task completion.
